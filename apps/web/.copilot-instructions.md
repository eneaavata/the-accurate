# GitHub Copilot Instructions for Accurate ERP

## Project Overview

This is a **multi-tenant ERP (Enterprise Resource Planning)** application built with:

- **Next.js 16** (App Router)
- **PWA (Progressive Web App)** support - must be mobile-friendly
- **TypeScript** for type safety
- **Tailwind CSS** for styling
- **Supabase** for backend and authentication
- **React Hook Form** with Zod validation
- **Lucide React** icons
- **Radix UI** components as base for UI components

## Core Principles

- **Mobile-First Design**: All UI components must be responsive and touch-friendly

### Authentication Patterns

**Protected Route Pattern** (for organization creation and other sensitive routes):

```tsx
'use client';

import { useEffect, useState } from 'react';
import { createClient } from '@/lib/supabase/client';
import { useRouter } from 'next/navigation';
import { User } from '@supabase/supabase-js';

export default function ProtectedPage() {
  const router = useRouter();
  const [user, setUser] = useState<User | null>(null);
  const [isLoading, setIsLoading] = useState(true);
  const supabase = createClient();

  useEffect(() => {
    checkAuthentication();
  }, []);

  const checkAuthentication = async () => {
    try {
      const {
        data: { user },
      } = await supabase.auth.getUser();

      if (!user) {
        router.push('/login');
        return;
      }

      setUser(user);
    } catch (error) {
      console.error('Authentication error:', error);
      router.push('/login');
    } finally {
      setIsLoading(false);
    }
  };

  if (isLoading) {
    return <div>Loading...</div>; // Or Skeleton component
  }

  if (!user) {
    return null; // Will redirect
  }

  // Protected content here
  return <div>{/* Your protected page content */}</div>;
}
```

### Proxy Authentication

The application uses Next.js proxy (`proxy.ts`) with Supabase middleware for server-side authentication:

- **Protected Routes**: `/`, `/setup`, and all organization routes require authentication
- **Public Routes**: `/login`, `/register`, `/offline` are accessible without authentication
- **Automatic Redirects**: Unauthenticated users are redirected to `/login`
- **Session Management**: Uses Supabase SSR for proper cookie handling via `updateSession` function
- **Static Assets**: Proxy bypasses authentication for static files and API routes

**Important**: When adding new protected routes, ensure they're not in the public paths list in `/lib/supabase/middleware.ts`.

### Database Operations

Always use Supabase client with proper typing:

```tsx
// For client components (singleton pattern - reuses same instance)
import { createClient } from '@/lib/supabase/client';

// For server actions and server components (creates new instance per request)
import { createClient } from '@/lib/supabase/server';

// IMPORTANT: Always await server client creation
const supabase = await createClient(); // Server-side
const supabase = createClient(); // Client-side (no await needed)

// Include organization filter for multi-tenancy
const { data, error } = await supabase.from('table_name').select('*').eq('organization_id', orgId);
```

**Security Note**: Never share server client instances between requests - each request must create a fresh instance to prevent session leakage.
